第一部分：背景与核查概述

1.1 背景与目的

为推进AI原生研发范式在全公司的落地，PMO组织对15个研发团队/部门进行项目上下文核查。本次核查覆盖三大目标、九项核心指标，旨在评估各团队在统一架构与规范、团队共识与遵从、AI可用性闭环方面的成熟度，识别短板并提供建议。

1.2 核查范围与方法

核查范围：15个研发团队/部门（详见部门编号对照表）。

核查方式：

  • 目标1（统一架构与规范）：以核查结果为准，已完成 Git 仓库文件抽检，结合问卷举证交叉核实。另参照 ai_lib.md Harness 六维模型评估维护成熟度（质量评分 0-10）。
  • 目标2（团队共识与遵从）：以核查结果为准，已逐项比对举证图片。
  • 目标3（AI可用性闭环）：以核查结果为准，已逐项比对举证图片。
